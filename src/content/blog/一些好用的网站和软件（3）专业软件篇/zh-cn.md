# 一些好用的网站和软件（3）专业软件篇

晚上十一点半。宿舍的灯关了，只有屏幕还亮着。

风扇在转。很轻，但在安静的时候你能听见它——一种持续的、没有起伏的白噪声，像这台机器在呼吸。我盯着桌面上那排图标，从左上角一直排到右下角，密密麻麻。有些我每天都点开，有些装了就再没动过，图标上落了一层看不见的灰。

我想说说这些图标。不是那种「工科生必备软件Top10」的说法。是它们真正待在我电脑里的样子——每一个都对应着某一次具体的卡住、某一段具体的时间、某个深夜里终于跑通之后我靠在椅背上听见自己呼吸的那几秒。

先说VSCode。

大一上学期我还在用Dev-C++。那个灰色的窗口，编译按钮要点两下，报错信息挤在底部一小条里，字很小。我写C语言作业的时候经常要凑近屏幕去看那个error到底报在哪一行。后来学长从后面经过，看了一眼，说，你装个VSCode吧。

装完打开的那个瞬间——怎么说——像是从一间只有一扇小窗的房间，搬进了一间四面都是玻璃的房子。代码高亮了，补全出来了，终端嵌在下面不用切窗口了。这些东西现在想起来很普通，但在那个晚上，我确实对着那个深色的编辑界面坐了一会儿，什么也没写，就是看着光标在那闪。

后来它就不只是编辑器了。Python、Markdown、Git、Remote SSH，插件一层一层叠上去，像在一块空地上慢慢盖房子。你现在让我回到Dev-C++那个灰窗口，我大概会有一种生理上的抗拒。由奢入俭难。这句话是真的。

Trae Solo是今年才装的。

字节做的一个AI编程工具。跟VSCode不一样的地方在于，它不是在一个已有的编辑器上面贴一层AI——它从骨架开始就是为「你跟AI一起写」这件事设计的。你直接用自然语言说你要什么，它帮你把项目搭出来。

我拿它试过一次，写一个串口数据可视化的小工具。从打字描述到程序跑起来，不到十分钟。那个感觉很奇怪。不是「哇好厉害」的那种兴奋，更像是一种轻微的不真实——我好像跳过了什么步骤。中间那段跟环境配置搏斗的、跟语法错误较劲的时间，被抽掉了。

但如果你已经会写代码，VSCode加Claude Code或者Cursor的组合上限更高。Trae Solo更像是一条捷径，适合你只想快速到达某个地方、不想在路上花时间的时刻。两条路都在，看你要去哪。

说到AI写代码这件事。Codex。

OpenAI今年放出来的这个东西，跟以前那个只做补全的Codex不是一个物种。你给它一个任务，它自己去读代码库、理解上下文、写、跑测试、修bug。整个闭环它自己完成。

我第一次让它干活是重构一个PID调参脚本。大一写的。变量名是a、b、c、temp1、temp2。注释没有。我自己打开看了两行就关掉了——不是看不懂，是不想看。那种面对自己半年前留下的混乱的抵触感，很具体。

把仓库丢给Codex。跟它说，变量名改成人能读的，加注释，逻辑别动。

三四分钟。它交了一版。

我打开看。每个函数上面有一段话说清楚这段在干嘛。核心算法没动。它只是把混乱理成了秩序。像有人帮你把一抽屉缠在一起的线一根根分开，用扎带绑好，放回去。抽屉还是那个抽屉，但你下次打开的时候不用再把手伸进去搅了。

当然它有时候会理解错。改着改着把不该动的地方也动了。所以我每次都会自己过一遍diff。信任这件事，跟验证不矛盾。
自从Claude code 曝出来有隐私问题后，我基本已经全面转向codex，接入了deepseek的模型。

好。下面这些东西，是电子人吃饭的家伙。

Keil uVision。

写51、写STM32、写ARM，绕不开它。第一次打开的时候我愣了一下——那个界面有一种很浓的属于2005年的气息。灰蓝色。工具栏挤了四五排。按钮很小，图标是那种像素感的旧风格。

但用久了你会发现，它丑，它稳。编译、下载、调试，一条龙。断点、单步、寄存器、内存。该有的全在。STC的官方例程和ISP工具跟它配合得最丝滑，所以我现在写STC32G还是用Keil。写STM32的话VSCode加PlatformIO也行，但Keil那个一键编译一键下载的无脑感——你不需要想任何事，点一下，灯就亮了，程序就进去了——这种不需要思考的顺畅，很难替代。

一个提醒。Keil是商业软件，正版很贵。我用的是破解版，具体怎么安装，不要问我，我也不知道，只记得当时学长交我们安装弄了一个下午，工科软件就是这样的，非常正常。

NI Multisim。

电路仿真软件。

三极管放大电路。你在纸上算，β值、静态工作点、增益，算了半页纸，得出一个数。但你心里没底。你不知道那个数是不是对的。你只是按照公式走了一遍流程，像填一张表格。

打开Multisim。画上去。信号源接输入，示波器接输出。点运行。

波形出来了。

增益对不对，相位偏了多少，截止频率在哪里——它不跟你解释，它只是把结果铺在你面前。绿色的线在黑色的背景上走，很安静。你看着那条线，心里那个悬了半天的东西就落地了。

它还有一个功能叫参数扫描。让一个电阻从1k变到100k，看输出怎么跟着变。在真实电路里你得换几十个电阻，拿万用表一个一个量。在这里你点一下鼠标。三秒。整个变化过程就展在你面前了，像延时摄影里一朵花开的全过程。

数电也一样。逻辑门、触发器、计数器，画上去就能跑时序图。期末复习的时候我好多电路都是先在Multisim里跑通了再进考场的。不是作弊。是确认。是你在纸上算完之后需要一个东西告诉你，对，你没有算错。

嘉立创。

严格来说它不是一个软件。它是一个平台。但它底下有一个嘉立创EDA，这个得说。

以前画PCB用Altium Designer。功能强。学习曲线也强。我光搞清楚怎么建一个元件封装就花了一整个下午，对着屏幕上一个灰色的矩形反复拖引脚，拖了删删了拖。而且AD正版好几万。学生用破解版，心里总归有一小块地方不太安稳。

嘉立创EDA不一样。免费的。在线的。打开浏览器就能画。不用装。不用配环境。不用对着一个安装向导等二十分钟。

而且它跟嘉立创的打样服务是通的。画完板子，一键下单。三天后PCB寄到。价格比一般的厂家要便宜，而且是顺丰包邮。

上学期智能车比赛我做扩展板，从画原理图到收到板子，不到一周。拆开那个牛皮纸信封的时候，板子边缘还有一点毛刺。我用指甲刮了一下。绿色的阻焊层下面，铜走线很细，比我画的线宽还要细一点。它从一块屏幕上的几何图形，变成了一块可以握在手里的、有重量的、有边缘会割手的东西。


我有时候觉得这件事的意义不只是「方便」。它把一道门槛拆掉了。以前你要做一块电路板，你得会AD，得有实验室的制板设备，得等。现在你只需要一个浏览器。

SolidWorks。

不是每个电子人都用得到。但如果你做过智能车、做过机器人、做过任何需要「把一个东西从脑子里拿出来放到桌上」的事情，它基本是绑定的。

我画过传感器支架、摄像头安装座、电池固定架。画完3D打印，装车上。尺寸不对，改一个参数，重新打。不用推倒重来。参数化设计的好处就在这里——你改一个数字，整个模型跟着你动。像一棵树，你拉一根枝条，其他的枝条会跟着调整自己的角度。

学习曲线比Keil和Multisim陡。我承认。刚开始拉伸、切除、配合的操作逻辑需要适应。但B站教程很多，跟着做两三个零件就能上手。对电子人来说不需要学到机械专业那个深度。能画个支架，能开个孔，能做个外壳。够了。

逐飞助手。

做过智能车的人知道摄像头调参是什么感觉。

MT9V03x接上板子。你在代码里改一个二值化阈值。编译。下载。跑一圈。看效果。不对。再改。再编译。再下载。再跑。这个循环。一圈大概五分钟。你改一个参数要五分钟才能看到结果。一晚上调不了几个参数。人会很烦躁。那种烦躁不是剧烈的，是钝的，是重复带来的磨损。

逐飞助手把这个循环打断了。摄像头接电脑，打开软件，画面实时出来。你调曝光、调增益、调阈值，所见即所得。不用编译，不用下载，不用等那五分钟。你拧一个旋钮，画面就跟着变。

它还看串口数据、看波形、看变量实时变化。我调PID的时候用它看速度曲线和转向曲线，比Keil那个调试窗口直观太多。

如果你也在打智能车。这个不装等于自废武功。我说的是真的。

然后是一些小的东西。体量不大。但用了就回不去。

Spipaste。剪贴板管理。

你可能觉得剪贴板就是Ctrl+C和Ctrl+V。但你有没有过这种时刻——从PDF里复制了一段话，切出去复制另一段，回来，第一段没了。它被覆盖了。你甚至不记得它原来长什么样。你只能回去重新找，重新复制。

Spipaste把复制历史全部存下来。快捷键呼出，想粘哪条粘哪条。它还有一个「贴图」功能——把一段文字或者一张图片钉在屏幕上。我写实验报告的时候把指导书的要求钉在左边，右边写。不用Alt+Tab来回切。

很小的东西。但就像你习惯了桌上有抽屉。再给你一块光板，你会觉得哪里不对。

CC Switch。多配置切换。很小众。

它最初是为Claude Code的多账号切换做的。如果你同时用多个AI工具，或者同一个工具有多套配置，它让你一键切过去。不用手动改环境变量。

我的场景是需要切换不同的token plan配置。以前每次换地方都要打开系统设置，找到环境变量，改，重启终端。烦。不是那种剧烈的烦，是每次都要重复同一个动作的、钝的烦。CC Switch把这个动作压缩成了一次点击。

如果你没有多配置的需求，你大概完全用不上它。但如果你有。你会觉得它是救命的。

Tabbit Browser。

一个浏览器。等等——浏览器有什么好推荐的。

你听我说。Chrome开了四五十个标签页的时候，上面那排图标已经挤成一条线了。你想找之前看的那个datasheet。你翻了半天。找不到。最后全关了重新搜。

Tabbit做标签页管理。分组、归档、搜索。你可以按项目分——「智能车」「模电」「摸鱼」——每组折叠起来，需要的时候展开。

我现在做项目用Tabbit开技术文档和datasheet，日常刷网页用Chrome。两个浏览器。分工明确。像桌上放两杯水，一杯是凉的白开水，一杯是热的茶。你不会搞混。

最后一个。UU远程。

远程控制软件。类似ToDesk、向日葵。但延迟和流畅度做得不错，对学生党友好。

我用它的场景很具体。实验室电脑上跑着Keil和一堆环境。我回宿舍了。想继续改代码。不用在宿舍电脑上再配一遍环境——那个过程要一个下午，而且总会少装一个什么东西。UU远程连回去。屏幕亮起来。实验室的桌面还在。Keil还开着。光标还停在我走之前停的那一行。

也拿来帮同学修过电脑。就是那个经典的「你帮我看看我电脑怎么了」。不用跑过去。连上去。两分钟。好了。

好了。十二个。

从写代码的到画电路板的。从仿真到建模。从剪贴板到远程控制。我对着屏幕数了一遍。有些每天都开，有些一周才点一次，有些——比如SolidWorks——可能一整个学期只在上次画支架的时候打开过。但它们都在那里。图标排着。安静的。等着被双击。

我大一刚来的时候，这台电脑的桌面是空的。干干净净。像一个刚交房的房间，什么都没有，什么都可以放。一年之后它变成了现在这样。每一个图标都是某一次「我需要做一件事」的残留物。是某一次卡住之后找到的出路。

它们把「想到」和「做到」之间的距离缩短了。以前做一块板子要学AD、找制板厂、等两周。现在五块钱三天。以前验证一个电路要去实验室抢面包板。现在打开Multisim，三分钟。

但我说不清这到底是一件好事还是一个需要被追问的事。门槛低了，路径短了。你更快地到达了。但路上那些跟环境配置搏斗的、跟报错信息较劲的、在面包板上插错线烧了一颗芯片然后坐在那发了一会儿呆的时间——那些时间是不是也是某种必要的东西。

我不知道。

风扇还在转。屏幕还亮着。走廊尽头有人经过，脚步声远了。

这些软件明天还会在桌面上。我明天还会点开其中几个。光标还会在那闪。

就这样吧。
